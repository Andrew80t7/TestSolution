##  Задача 1. Интеграция с Ozon Seller API (v3)

### 1.1 Описание и бизнес-логика
Скрипт выполняет авторизацию в Ozon Seller API и пошагово собирает актуальные данные о товарах:
1. Запрашивает список идентификаторов (`product_id`) товаров магазина через эндпоинт `/v3/product/list`.
2. Собирает детализацию (название, артикул, цены, складские остатки) пакетами (по 50 товаров) через эндпоинт `/v3/product/info/list`.
3. Анализирует остатки каждого товара и сравнивает их с заданным порогом `LOW_STOCK_THRESHOLD`.

### 1.2 Работа с документацией и ИИ
* **Передаваемый контекст:** Структура заголовков авторизации Ozon (`Client-Id`, `Api-Key`), а также требуемые поля ответа (`offer_id`, `price`, `stocks`).
* **Формулировка запросов:** Модели задавался запрос на написание модульной функции с разделением на получение ID товаров и получение детальной информации, с обязательной обработкой ошибок HTTP (`raise_for_status()`) и поддержкой пагинации по `last_id`.

### 1.3 Проверка методов на актуальность
* **Проверка эндпоинтов:** ИИ изначально мог предложить устаревшие методы API `v1/product/info` или `v2/product/info/list`. Была проведена сверка с официальной документацией Ozon Seller API: методы `v1` и `v2` помечены как deprecated. Для работы взят актуальный метод `/v3/product/info/list`.
* **Проверка структуры ответа:** Метод `/v3/product/info/list` возвращает информацию об остатках внутри объекта `stocks` (разбитого на `present` и `reserved`). Скрипт корректно суммирует доступные единицы товара.

### 1.4 Фактический вывод работы с Ozon API

# Задача 2. Система алертинга в Telegram

## 2.1 Описание и сетевая архитектура

Скрипт генерирует форматированный HTML-отчет и отправляет его в Telegram-чат.

В процессе работы возникли сетевые ограничения:

### Проблема IPv6

Ошибка:

```text
[Errno 101] Network is unreachable
```

Причина: попытки `urllib3` отправлять запросы через неактивный IPv6-интерфейс Linux.

**Решение:** внедрена принудительная инициализация сокетов через `socket.AF_INET` (IPv4).

### Блокировка api.telegram.org

Прямое подключение приводило к `Timeout`.

**Решение:** настроен Cloudflare Worker:

```text
https://delicate-scene-fbf2.vysotnik3.workers.dev
```

Worker выполняет роль прокси. Запросы из Python отправляются на Worker, который перенаправляет их в Telegram API.

# Задача 3. Очистка и нормализация каталога (`catalog_raw.csv`)

## 3.1 Описание ETL-процесса

Скрипт `clean_catalog.py` принимает файл выгрузки `catalog_raw.csv` и приводит его к структурированному виду в `catalog_clean.csv`.

### Основные этапы

- Очистка цен и приведение к единому типу `float`.
- Извлечение характеристик из названия (Бренд, OEM-номер, Фасовка).
- Фильтрация мусора, пустых значений и дубликатов.

---

## 3.2 Анализ крайних случаев и неоднозначности данных

| Неоднозначность / крайний случай | Пример из исходных данных | Логика обработки и решение |
|----------------------------------|---------------------------|----------------------------|
| Разнородные форматы цен | `"1 500 руб"`, `"1500.00"`, `"1500р"`, `"1 500 rub"`, `"1 500,00"` | Используется `re.sub(r'(\d)\s+(\d)', r'\1\2', s)` для удаления пробелов между цифрами, замена запятой на точку и извлечение регуляркой `r'\d+(?:\.\d+)?'`. Результат конвертируется в `float`. |
| Скрытые пробельные символы | Неразрывные пробелы `\xa0` в строках цен | Предварительная очистка через `.replace('\xa0', ' ')`. |
| Разные форматы OEM-номеров | `90919-01253` (с дефисом) и `0242236564` (сплошной) | Шаблон `\b([A-Z0-9]{3,}(?:-[A-Z0-9]+)+\|[A-Z0-9]{6,15})\b` извлекает как дефисные артикулы, так и длинные буквенно-цифровые коды автопроизводителей. |
| Неявное указание фасовки | `"Свеча зажигания (комплект)"`, `"Свеча зажигания 1шт"` | Поиск ключевых слов (`комплект`, `компл`, `набор`, `set`, `пачка`) через `re.IGNORECASE`. При совпадении присваивается значение **Комплект**, иначе — **Штука**. |
| Определение бренда | Бренд может быть в начале, середине или отсутствовать | Используется словарь известных автобрендов (`BOSCH`, `NGK`, `DENSO`, `TOYOTA` и др.). Если совпадений нет, берется первое текстовое слово из букв длиной от 2 символов. |
| Дубликаты и пропуски | Полностью пустые строки или повторяющиеся записи | Автоматическое удаление через `df.dropna(how='all')` и `df.drop_duplicates()`. |
